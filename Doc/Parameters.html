Introduction

Parameters allow to control options of the player

Loading

"loading":
'onclick': displays the splash screen and waits for the user to click to start loading the content.
'preload': displays the splash screen and starts to load the content in the background.
'autoplay': (default) automatically launches the player.

"progressiveLoading": if true displays each object on the screen as soon as loaded; else (default) wait that all the media is loaded before displaying it.

"showProgressBar": If true (default) display a progress bar during loading. If false, no progress bar is displayed.

Splash screen customization

In Web Component Tag parameters
'waitloading': If true, the splash screen stays visible until loading is finished.
'splashscreenImage': Allows to define a custom image (jpg or png). Path must be absolute or relative to the webComponent js file.

In createBundle method of Player.js
“splashScreen” variable:Allow to control what is displayed and how during loading
'waitloading': If true, the splash screen stays visible until loading is finished.
'custom': Allows to define a custom background for the splash screen. It can be a string containing the html code for the background or directly a DomObject. 


    let splashscreen =
    {
    "custom": "",
    "waitloading": true
    }
    let splashscreenstr = JSON.stringify(splashscreen);

Player

"play": If true (default) starts to play the experience at the end of loading.
"transparentBackground": If true displays no background for the player (default value = false).
"visuOptionsPreset": Set visualisation engine with pre-configured options (Low/Medium/High).

These options are useful only for media published from Creative Experience.

Display

"showReferenceAxisSystem": If true displays an axis system at the bottom right of the player (default value = false).
